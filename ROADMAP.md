# Claude Dashboard 升级路线图

本文档记录 Claude Dashboard 的未来开发计划和功能升级路线。

## 当前状态

**已实现的基础功能：**
- ✅ 项目列表浏览
- ✅ 会话列表查看
- ✅ 会话详情（消息流）
- ✅ 基础搜索功能
- ✅ 书签管理
- ✅ 标签系统
- ✅ Markdown 导出
- ✅ 实时文件监控
- ✅ 代码高亮（Prism.js）
- ✅ Markdown 渲染（表格、列表、代码块等）
- ✅ 服务端搜索（高性能）
- ✅ 搜索高亮显示
- ✅ 高级搜索筛选（项目、角色、日期）
- ✅ Thinking/Tool Result 折叠
- ✅ 搜索结果 Drawer（右侧弹窗查看会话）
- ✅ 项目卡片增强（头像、可编辑描述、打开文件夹）
- ✅ Markdown 文档查看器（项目中 .md 文件预览，智能过滤非项目文档）
- ✅ 文档浏览器宽度加倍优化
- ✅ 项目名称显示优化 - 自动提取代码文件夹名称作为项目名
- ✅ Git 集成 - 显示项目分支、提交历史和变更统计
- ✅ Session Drawer - 在项目详情页以抽屉形式展示会话，保持浏览连贯性

## 第一阶段：体验优化（近期）

### 1.1 会话管理增强
- [x] **会话恢复** - 一键复制 `claude --resume <session-id>` 命令
- [x] **会话重命名** - 允许用户为会话设置自定义名称
- [ ] **会话删除** - 从界面上删除不需要的会话（仅删除记录，不删文件）
- [ ] **会话归档** - 将不常用的会话归档，减少列表噪音
- [ ] **批量操作** - 批量删除、批量添加标签

### 1.2 消息展示优化 ✅
- [x] **代码高亮** - 消息中的代码块支持语法高亮（使用 Prism.js + oneDark 主题）
- [x] **Markdown 渲染** - 完整支持 Markdown 格式（表格、任务列表、代码块、引用等）
- [x] **折叠/展开** - Thinking 内容默认折叠，Tool Result 长内容可折叠
- [ ] **图片展示** - 支持显示消息中的图片内容
- [x] **消息跳转** - 从搜索结果直接跳转到消息位置，自动高亮匹配内容
- [x] **会话内搜索** - 在会话详情页搜索消息，支持上下导航和快捷键

### 1.3 搜索功能完善 ✅
- [x] **服务端搜索** - 搜索逻辑移到服务端，避免客户端处理大量数据
- [x] **搜索高亮** - 结果中高亮显示匹配的关键词（黄色标记）
- [x] **高级搜索** - 支持按项目、角色、时间范围筛选
- [ ] **搜索历史** - 保存最近的搜索记录
- [ ] **正则搜索** - 支持正则表达式搜索

### 1.4 性能优化 ✅
- [x] **搜索性能** - 服务端搜索 + 结果限制（最多50个会话）+ 防抖处理
- [x] **服务器端缓存** - 30秒缓存机制，避免重复扫描文件系统
- [x] **单例扫描器** - 复用扫描器实例，防止重复创建
- [x] **按需加载** - 查询单个项目时只扫描目标项目
- [x] **前端请求防抖** - 30秒内不重复请求项目列表
- [x] **并发扫描保护** - 防止同时发起多个扫描请求
- [ ] **虚拟滚动** - 大数据量下的消息列表虚拟滚动
- [ ] **懒加载** - 会话消息分页加载，不一次加载全部
- [ ] **搜索索引持久化** - 将搜索索引保存到本地，减少重建时间

## 第二阶段：功能扩展（中期）

### 2.1 数据分析
- [ ] **使用统计** - Token 使用量统计（日/周/月）
- [ ] **成本分析** - 按项目、时间维度分析 API 调用成本
- [ ] **会话时长** - 统计每个会话的持续时间

- [ ] **评论系统** - 对消息添加评论
- [ ] **会话模板** - 保存常用会话模式作为模板

### 2.4 导入导出
- [ ] **多种格式导出** - 支持 JSON、HTML、PDF 导出
- [ ] **批量导出** - 导出整个项目或多个会话
- [ ] **导入功能** - 从备份文件恢复书签、标签数据

## 贡献指南

欢迎提交 PR 实现路线图中的功能！

### 优先级标签
- `priority-high` - 高优先级，近期实现
- `priority-medium` - 中优先级，中期规划
- `priority-low` - 低优先级，长期规划
- `good-first-issue` - 适合新手上路

### 提交规范
1. 在 Issue 中讨论功能设计
2. 提交 PR 时关联对应的 Issue
3. 确保通过所有测试
4. 更新相关文档
